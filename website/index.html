Changelog

3.0.2 - Error handling

Up unitl now, every time the proxy server had issues users were presented with never ending loading screens on Crunchyroll. This should be handled better now.

  • Added a notification that shows up on incorrect proxy settings.
  • Added a fallback when the proxy runs into a timeout to not use the proxy.

3.0.1 - A housekeeping update

This release fixes some bugs and removes the code used for "old crunchyroll".

  • Removed legacy code
  • Do not display password for custom proxy configuration
  • Fix checkbox not displaying correct setting for custom proxy configuration
  • Use DNS of proxy service

3.0.0 - CR-Unblocker turns 3!

CR-Unblocker now supports the new Crunchyroll Beta. This means no longer being limited to non logged in usage.

  • Proxy requests from new token endpoint using a SOCKS proxy.
  • Add options to set a custom proxy configuration in extension settings.
  • Remove logic handling user logins as we don't need to do that anymore.
  • Modernize some code paths for Crunchyroll Classic.

2.3.1

Most issues seem to be errors getting new sessions. I think this is largely related to the reload loop. To improve on this I implemented a timer to only reload once a minute - if the session can not be obtained, it is unlikely to work a few seconds after.

  • Added timer between session reloads

2.3

Good news: unblocking while logged in is back!

Some of you have hinted me a way but I could not replicate it for some reason. Thanks for telling me, and sorry for not getting this working sooner :)

  • Fixed login handling
  • Added disable button to popup to quickly disable the addon

2.2.3

This is just a small update, sorry still no solution for logged in users :(

But I removed the unblock servers that do not work anymore, this should make switching faster.

Also, region switching won't happen while logged in. This way you can login/logout depending on the anime you want to watch. Still inconvenient, but better than nothing!
